# /batch — 批量处理小任务

## 用途

批量处理多个小任务，但必须分批、可控、可回滚。

## 适用场景

- 批量补充 loading/empty/error 状态
- 批量修复 lint 警告
- 批量添加类型标注
- 批量处理 TODO 注释
- 批量整理文档格式

## 示例

```
/batch 给所有页面补充 loading 和 error 状态
/batch 修复 frontend tsc --noEmit 的所有报错
/batch 把所有 TODO: 改成具体任务描述
```

## 执行流程

1. 拆解成小任务
2. 输出 Batch Plan 表格（序号/任务/文件/风险/是否执行）
3. 只自动执行低风险任务
4. 中高风险暂停等待确认
5. 每批完成输出变更摘要
6. 全部完成输出 Batch Report

## 限制

- 不一次改全项目
- 不跨架构层乱改
- 不改数据库/鉴权/核心业务
- 不静默修改

## 风险等级

中等偏高 — 批量操作可能影响面广，必须分批审核。
