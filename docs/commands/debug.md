# /debug — 系统化排查 Bug

## 用途

系统化排查 bug：先收集上下文 → 提出多个假设 → 验证定位根因 → 最小修复。

## 适用场景

- 页面白屏/闪退
- 测试失败
- API 返回异常（422, 500）
- 数据与预期不符
- 前端构建失败
- TypeScript 类型报错
- AI 修改后项目跑不起来

## 示例

```
/debug 总览页面闪退
/debug pytest app/tests/test_agent.py - 11 failed
/debug /api/research/theses 返回 422
```

## 执行流程

1. 收集上下文（日志、最近改动、测试结果）
2. 提出 3 个可能原因，按概率排序
3. 为每个原因提供验证方法
4. 最小验证定位
5. 最小修复
6. 运行测试
7. 输出 Debug Report

## 限制

- 不一上来就大改
- 不猜测式修复
- 不删除测试
- 不吞掉异常

## 风险等级

中等 — 修复通常很小，但定位过程需要谨慎。

## 常用第一步

```
/debug 看一下 backend/backend.log 最近 50 行
```
