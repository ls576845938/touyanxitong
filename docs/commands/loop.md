# /loop — 多轮闭环迭代

## 用途

让 Claude Code 在任务上形成"计划 → 执行 → 测试 → 审查 → 修复 → 记录"的闭环，最多 5 轮。

## 适用场景

- 一个功能需要反复迭代
- 测试失败需要自动修复
- 页面效果需要多轮优化
- 重构后需要回归验证

## 示例

```
/loop 修复 test_agent.py 中 11 个失败测试
/loop 让 Dashboard 页面加载时间降到 2 秒内
/loop 补齐 thesis_review_engine 的边界测试
```

## 执行流程

每轮: Plan → Act → Test → Review → Fix → Record

停止条件:
- 任务完成
- 连续 2 轮无进展
- 遇到高风险修改
- 达到 5 轮上限

## 限制

- 不无限循环
- 不扩大范围完成任务
- 不盲目重写
- 不掩盖测试失败

## 风险等级

中等偏高 — 多轮修改可能累积风险，每轮必须审查。
